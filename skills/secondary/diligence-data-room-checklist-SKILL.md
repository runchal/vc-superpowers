# Diligence Data Room Checklist

## Pre-work (MANDATORY)
1. Read `vc-work/context.md` to understand accumulated context
2. Ask user for current data room inventory and gaps → append to `context.md`
3. Log session start in `vc-work/session-logs/YYYY-MM-DD-HH-MM.md`

## Contents (by folder)
- **Product**: demo video, roadmap, architecture (1 page), security notes.
- **Customers**: ICP, pipeline, pilots→PO conversion metrics, churn/retention.
- **Financial**: P&L, cash plan, unit econ model, pricing policy, ARR bridge.
- **Ops**: vendors, supply chain map, QA, warranties/SLAs.
- **Legal**: cap table, IP, contracts, certifications, insurance.

## Common Mistakes
- Version chaos; missing ARR bridge; pilot terms not standardized.
